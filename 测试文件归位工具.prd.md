# 需求文档: 测试文件归位工具 (代码测试同目录化)

## 1. 目标
强制要求项目中所有**已经存在**的单元测试文件必须与其对应的源代码文件存放在**同一个目录**下。

本工具用于扫描指定目录，发现那些“身首异处”的测试文件，并引导 AI Agent 将其“归位”。通过让测试与代码 Side-by-Side，利用物理位置的唯一性解决通用文件名（如 `views.py`）导致的全局匹配冲突。

## 2. 归位规则
*   **归位目标**：源代码 `path/to/文件名.py` 对应的测试必须位于 `path/to/test_文件名.py`。
*   **寻找来源**：工具会全局搜索名为 `test_文件名.py` 的文件。
*   **归位执行规范**：
    *   **强制使用 Git**：AI Agent 必须使用 `git mv` 命令执行归位操作，以保持 Git 历史记录的连贯性。
    *   **严禁直接删除**：严禁使用 `rm` 或其他删除手段处理匹配的测试文件，必须通过移动（Move）完成归位。
*   **核实与占位逻辑**：
    *   AI Agent 发现全局存在同名测试时，必须核实其内容。
    *   **匹配**：执行归位（移动并适配）。
    *   **不匹配 (误报)**：AI Agent 必须在源代码同级目录创建一个**本地占位测试文件**。
*   **占位文件规范**：
    *   文件名：`test_文件名.py`。
    *   内容：**严禁编写任何 import 或测试逻辑**。仅允许包含一行注释：`# 为避免归位误报自动生成的占位测试`。
    *   作用：通过物理文件的存在，使工具在下次运行时判定该文件“已对齐”，从而彻底消除该条误报。

## 3. 使用方法
```bash
uv run 测试文件归位工具.py <要整理的源代码目录> [--verbose]
```

## 4. 输出规范
### 4.1 状态 A: 已对齐 (成功)
*   **提示**: "[归位检查]: 成功。该目录下所有已发现的测试文件均已与其源代码同级。"

### 4.2 状态 B: 未对齐 (需要归位)
*   **提示**: "[归位检查]: 发现位置错误的测试文件。"
*   **动作指引**:
    1. **核实并移动**：如果内容匹配，**必须使用 `git mv`** 移动该测试文件到源代码同级目录并适配导入。严禁直接删除（rm）文件。
    2. **创建占位文件**：如果内容**不匹配**（确认为误报），请立即在 `归位目标位置` 创建仅含注释 `# 为避免归位误报自动生成的占位测试` 的文件。

### 4.3 详情模式 (--verbose)
*   启用后，工具将额外输出：
    *   **已跳过的目录/文件**（基于排除规则）。
    *   **已确认对齐的文件**。
    *   **正在扫描的路径**。

## 5. 关联脚本
配套脚本为：`测试文件归位工具.py`
