# 通用开发规范 (Common PRD)

本项目的所有检查工具和脚本必须严格遵守以下通用规范：

## 1. 核心约束
- **零外部依赖**：仅允许使用 Python 标准库（Standard Library）。严禁引入任何第三方类库（如 `requests`, `pandas`, `pydantic` 等）。
- **统一命令行接口**：所有脚本必须接受且仅接受一个参数，即需要检查的目标目录路径。
- **默认行为**：如果未提供参数，脚本应默认检查当前目录（`.`）。
- **排除隐藏目录**：在递归扫描目标目录时，必须主动跳过所有以点（`.`）开头的隐藏目录（如 `.git`, `.venv`, `.vscode`, `.idea` 等）。

## 2. 命名与结构
- **中文命名**：所有功能脚本及其对应的测试脚本文件名必须使用中文。
- **测试必备**：每一个功能脚本必须配备相应的测试脚本，命名规范为 `test_<脚本名>.py`。
- **文档自包含**：脚本头部的 docstring 必须包含其对应的 PRD 内容，并明确引用本通用规范。

## 3. 输出与交互
- **退出码规范**：检查通过或执行成功时以 `0` 退出；发现不合规项、执行失败或需要 AI Agent 介入时以 `1` 退出。
- **动作指引**：当脚本以 `1` 退出时，必须打印清晰、步骤化的 AI Agent 动作指引。
- **语言要求**：所有输出指引和沟通必须使用中文。**最后，请 AI Agent 使用中文回答。**
