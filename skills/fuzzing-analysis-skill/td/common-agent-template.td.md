# 通用 Agent 模板技术文档

## 1. 目标

- 提供一层可复用的通用 Agent 基础能力。
- 让不同 Agent 以统一方式接入运行入口、参数处理、日志、状态回传和结果输出。
- 将基础能力从具体 Agent 中剥离出来，避免重复实现。

## 2. 范围

- 本文档只描述通用 Agent 模板本身。
- 不描述某个具体 Agent 的业务逻辑。
- 不描述测试补齐任务的分析、执行细节。
- 不负责替代具体业务 skill 的职责。

## 3. 目录约定

- 通用 Agent 模板不作为 skill 存在，而是作为一份共享公共能力放在仓库内的独立目录中。
- 每个 skill 可以保留自己的 `scripts` 文件夹，但其中只放本 skill 的私有脚本、适配逻辑和调用入口。
- 通用 Agent 模板由所有 skill 共享，不在各个 skill 目录中重复拷贝。
- 各个 skill 的脚本通过稳定的相对路径或统一入口调用这份公共模板。

## 4. 依赖说明

- 本文件只描述通用 Agent 模板的实现层，不重复定义所有 skill 共享的公共协议。
- 所有 skill 共享的接入协议、参数协议、输出协议、状态协议、日志协议和兼容性协议，统一放在 [`common-agent-contract.td.md`](../common/common-agent-contract.td.md)。
- 模板实现必须遵守公共协议文件中的约定。

## 5. 实现模块

- 协议读取模块：负责读取 `SKILL.md` 并提取调用所需的元信息。
- 参数校验模块：负责验证 `payload` 的结构、版本号和必填字段。
- 工作目录模块：负责创建和管理 `.agents_workdir` 下的任务目录。
- 日志模块：负责初始化日志文件、写入运行事件和保存参数快照。
- 状态封装模块：负责将执行过程映射为统一的 `status` 和 `next_action`。
- 结果封装模块：负责将业务输出包装为统一的 `result` 结构。
- 错误适配模块：负责把参数错误、运行错误和外部依赖错误转换为结构化结果。
- 兼容性检查模块：负责检查 `schema_version` 和 `skill_version` 的兼容关系。
- 公共模板实现必须采用面向对象方式编写，公共能力应封装为类，不要拆成一组松散函数。
- 推荐的类结构包括 `TemplateRunner`、`SkillContractReader`、`PayloadValidator`、`WorkdirManager`、`LogRecorder`、`StatusMapper`、`ResultBuilder` 和 `ErrorAdapter`。
- `TemplateRunner` 负责串联整体流程，其他类分别负责协议读取、参数校验、目录管理、日志、状态、结果和错误转换。

## 6. 接入方式

- 通用模板由各个 skill 的入口文件调用。
- skill 自身只实现业务逻辑，不重复实现协议解析、目录管理、日志和结果封装。
- 业务入口和公共模板之间通过共享协议文档保持一致。
